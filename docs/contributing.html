开发

我们非常欢迎您加入 BANetworking 的开发，这份文件将帮助您了解开发流程。

开始之前

请遵守以下准则：

  • 遵守 objective-c-style-guide 代码规范。
  • 使用 4 个空格缩排。
  • 不要把逗号放在最前面。

工作流程

  1. Fork [beyondabel/BANetworking]

  2. 把库（repository）复制到电脑上，并安装所依赖的插件。

  3. 新增一个功能分支。

  4. 开始开发。

  5. 推送（push）分支。

  6. 建立一个新的合并申请（pull request）并描述变动。

更新文档

文档都会在 BANetworking文档进行更新。

反馈问题

当您在使用 BANetworking 时遇到问题，您可以尝试在 问题解答 中寻找解答，或是在 GitHub 上提问。如果你没有找答案，请在 Github 报告它。
